# Code of Conduct

## Our Standard

Lattice project spaces should stay technical, direct, and respectful.
Participants are expected to:

- focus criticism on code, design, tests, documentation, and behavior;
- make room for clarifying questions and corrections;
- avoid personal attacks, harassment, and discriminatory language;
- respect maintainer decisions about scope, release safety, and security
  handling.

## Unacceptable Behavior

Unacceptable behavior includes harassment, threats, sexualized language or
imagery, sustained disruption, doxxing, impersonation, and publishing private
security details before coordinated disclosure.

## Enforcement

Maintainers may edit or delete comments, close discussions, block users, or
restrict repository access when behavior makes the project harder or unsafe to
maintain.

Report conduct concerns through GitHub private vulnerability reporting when the
report includes private personal data or safety-sensitive details:

<https://github.com/ElevatedDev/Lattice/security/advisories/new>

For non-sensitive conduct issues, open a repository issue with links, dates, and
a concise description.

## Scope

This code of conduct applies to repository issues, pull requests, discussions,
reviews, release artifacts, and other project-controlled spaces.
